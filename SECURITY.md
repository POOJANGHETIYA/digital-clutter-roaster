# Security Policy

## Supported versions

This project is currently in active development.
Only the latest `main` branch is supported for security fixes.

## Reporting a vulnerability

Please do not open public issues for potential vulnerabilities or privacy flaws.

Report privately using one of the channels below:

1. GitHub private vulnerability reporting (preferred)
2. Email: swe.poojan@gmail.com (for reporters who cannot use GitHub advisories)

- GitHub private advisory link: https://github.com/swepoojan/digital-cluster-roaster/security/advisories/new

- A clear description of the issue

- Affected environment (OS, browser, app commit/tag)
- Potential impact
- Any proof-of-concept details that help reproduce safely

## Disclosure expectations

- Please keep vulnerability details private until maintainers triage and fix the issue
- Avoid posting exploit details in public issues, discussions, social media, or blog posts before coordinated disclosure
- Maintainers will coordinate a disclosure timeline once a fix is available

## What to expect

- Initial acknowledgment target: within 5 business days
- Triage and severity assessment after acknowledgment
- Coordinated disclosure once a fix is available

## Privacy-sensitive reports

Because this app analyzes local file metadata, never share raw personal file names or paths publicly.
If examples are needed, redact personal identifiers before sending logs or screenshots.
